# OC — самый первый шаг к своей ОС

Это учебный старт собственной ОС: C++ программа собирает загрузочный BIOS-образ (`build/os.img`) без NASM/GRUB/кросс-компилятора. Образ можно запустить в QEMU из UCRT64.

Теперь проект уже разделён на маленький bootloader и отдельное real-mode ядро. Boot sector остаётся 512-байтным, загружается BIOS-ом по адресу `0x7C00`, читает дополнительные секторы с диска через BIOS `int 13h` и передаёт управление ядру по адресу `0x8000`. Благодаря этому shell ядра больше не ограничен одним загрузочным сектором.

## Требования

- `g++` из UCRT64/MinGW или другой C++17 компилятор
- `make` (опционально, но удобно)
- `qemu-system-i386`

## Быстрый запуск в UCRT64

```sh
make run
```

Если `make` недоступен, выполните команды вручную:

```sh
mkdir -p build
g++ -std=c++17 -O2 -Wall -Wextra -pedantic src/build_image.cpp -o build/build_image
./build/build_image build/os.img
qemu-system-i386 -drive format=raw,file=build/os.img
```

На экране QEMU должно появиться меню ядра:

```text
OC kernel shell
===============
H help A about C home R reboot
E echo B beep M memory K keyboard
D disk T tasks V video S syscalls
N notes P processes F color G goto X pmode

Press a key...
```

Доступные клавиши:

- `H` — показать справку.
- `A` — показать информацию о ядре.
- `C` — очистить экран и вернуться в главное меню.
- `R` — перезагрузить виртуальную машину через BIOS.
- `E` — перейти в echo-режим: ядро печатает введённые символы, `Esc` возвращает в меню.
- `B` — подать звуковой сигнал через BIOS teletype/BEL.
- `M` — показать карту памяти v0.
- `K` — показать заметки о клавиатурном вводе.
- `D` — показать информацию о дисковом загрузчике.
- `T` — показать заглушку менеджера задач.
- `V` — показать сведения о видеоконсоли.
- `S` — показать план системных вызовов.
- `N` — показать заметки о следующих шагах ядра.
- `P` — показать заглушку таблицы процессов.
- `F` — переключить цвет текста на следующий VGA foreground color.
- `G` — показать демо позиционирования текста в видеопамяти.
- `X` — перейти в protected mode demo: shell остановится, а 32-битный код напишет сообщение напрямую в `B8000`.

## Что здесь происходит

`src/build_image.cpp` генерирует два компонента образа:

1. **Bootloader** — первый 512-байтный сектор с сигнатурой `0x55AA`. BIOS загружает его по адресу `0x7C00`. Он сохраняет boot drive из `DL`, читает секторы ядра начиная с сектора 2 через BIOS `int 13h` и прыгает на `0000:8000`.
2. **Kernel** — отдельный real-mode код, выровненный по 512-байтным секторам. Он инициализирует сегменты/стек, очищает экран прямой записью в видеопамять `B800:0000`, ведёт программную позицию курсора, прокручивает экран при переполнении, синхронизирует hardware cursor через VGA-порты `0x3D4/0x3D5`, поддерживает смену VGA-цвета текста и позиционирование вывода. Интерактивный shell пока остаётся real-mode shell: он читает клавиатуру через BIOS `int 16h` и перезагружает машину через BIOS `int 19h`. Команда `X` запускает отдельный one-way protected mode demo через GDT и `CR0.PE`; после переключения BIOS-прерывания больше не вызываются.

Protected-mode переход вынесен в `src/kernel/protected_mode.hpp`: там явно находятся GDT, `lgdt`, установка `CR0.PE` и far jump в 32-битный code segment. Подробнее о текущем разделении real-mode shell и protected-mode demo см. `src/kernel/README.md`.

Обработчик клавиш нормализует латинские буквы, поэтому команды работают и в нижнем, и в верхнем регистре.

## Следующие шаги

1. Вынести bootloader и kernel в отдельные исходные модули.
2. Вынести консоль в отдельный модуль и добавить удобные API `put_at`, `set_color`, `scroll`.
3. Перенести основной kernel shell в protected mode вместо одноразового demo-перехода.
4. Скомпилировать freestanding C++ kernel и передавать ему управление.
5. Добавить IDT, таймер IRQ и настоящий планировщик задач.
