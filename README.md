# DOC — самый первый шаг к своей ОС

Это минимальный учебный старт: C++ программа собирает загрузочный BIOS-образ (`build/os.img`) без NASM/GRUB/кросс-компилятора. Образ можно запустить в QEMU из UCRT64.

Пока это не полноценное ядро, а базовая интерактивная BIOS-среда: boot sector загружается BIOS-ом, очищает экран, печатает меню и реагирует на клавиши. Это самый простой фундамент, от которого можно идти к загрузчику, protected mode и настоящему C++ kernel.

## Требования

- `g++` из UCRT64/MinGW или другой C++17 компилятор
- `make` (опционально, но удобно)
- `qemu-system-i386`

## Быстрый запуск в UCRT64

```sh
make run
```

Если `make` недоступен, выполните команды вручную:

```sh
mkdir -p build
g++ -std=c++17 -O2 -Wall -Wextra -pedantic src/build_image.cpp -o build/build_image
./build/build_image build/os.img
qemu-system-i386 -drive format=raw,file=build/os.img
```

На экране QEMU должно появиться мини-меню:

```text
OC mini shell
------------
H help  A about
C clear R reboot

Press a key...
```

Доступные клавиши:

- `H` — показать справку.
- `A` — показать информацию об ОС.
- `C` — очистить экран и вернуться в главное меню.
- `R` — перезагрузить виртуальную машину через BIOS.

## Что здесь происходит

`src/build_image.cpp` создаёт ровно 512 байт загрузочного сектора. BIOS загружает эти 512 байт по адресу `0x7C00`, проверяет сигнатуру `0x55AA` в конце сектора и передаёт управление коду. Код использует BIOS interrupt `0x10` для вывода текста, `0x16` для чтения клавиатуры и `0x19` для перезагрузки.
